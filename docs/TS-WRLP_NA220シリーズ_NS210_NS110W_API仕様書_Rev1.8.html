  • 本仕様書のExampleのURLはTS-WRLP仕様となっています。TS-NA220シリーズ/TS-NS210/TS-NS110Wをご利用の場合は/1/camera-cgiのように/1を追加してリクエストしてください。(/snapshot.cgiなどルートディレクトリでアクセスできるものを除く)

  • 視聴以外の機能は管理者ユーザーのみ利用可能です。

  • httpsでは自己証明書を利用しています。接続の際はクライアントの証明書検証は無視するなどの対応をお願いします。

+ Parameters

Parameters 書式 ¶

  • optionalの記載があるパラメータは省略可能です。省略した場合はDefaultの値が採用されます。

  • 値の候補が無いパラメータは、Exampleの値を固定値として採用します。

  • - で繋がれた値は範囲を示します。

ResponseはAPIの応答を定義しています。

+ Response (HTTPCODE)

Response 要素 ¶

Headers - ヘッダー
Body - ボディ
Schema - ボディの書式

共通 仕様 ¶

POSTリクエストの実行結果は返しません。レスポンスコードはHTTP 200のみ、Bodyは空文字列になっています。
引数間違いや不正な値、機種により対応していない機能を指定しても、エラーコードなどは返答しません。無効な設定が指定された場合は、そのリクエストは無視され元の値を維持します。

リクエスト例 ¶

curl -vX POST --digest --user admin:1qaz http://192.168.1.8:80/camera-cgi/admin/param.cgi -d “action=update&Network_networkType=0&Network_IP=TEST&Network_netmask=255.255.255.0&Network_dummy=192.168.1.1”

audio/transmit.cgi  ¶

通話
GET/camera-cgi/audio/transmit.cgi

対象: TS-WRLP,TS-NA220 Series,TS-NS210,TS-NS110W
通話機能の音声を送信します。

Caution ¶

  • 接続後、クライアントは応答のヘッダに含まれている文字列を確認してからBodyを送信してください。

  • bitrateやcodecが合わないデータを送った場合、無音またはノイズが出力されます。ノイズは意図しない大音量を生じる場合がありますのでご注意ください。カメラ内部のアンプやスピーカーを損傷する恐れがありますので、ノイズが生じた場合は速やかに音声データの送信を中止してください。

<対応する音声形式>

Product AAC GSM
TS-WRLP No Yes
TS-NS210,NA220(W) Yes No
TS-NS110W Yes No

<クライアントからの使用例>

  1. カメラの[HttpPort]にSocket接続

  2. Socket に次を送信

    GET /camera-cgi/audio/transmit.cgi HTTP/1.1\r\nConnection: Keep-Alive\r\nCache-Control: no-cache\r\n\r\n\r\n

  3. 401 Unauthorized の応答があった場合、Digest 認証情報を付与して再送

    GET /camera-cgi/audio/transmit.cgi HTTP/1.1\r\nConnection: Keep-Alive\r\nCache-Control: no-cache\r\nAuthorization: Digest username=“admin”, realm=“Network Camera”, nonce=3f16834beee735c795b53a4d25d5fa50, uri="/camera-cgi/audio/transmit.cgi", response=“d44876fe10cb98d9ee101ca527126412”, qop=“auth”, nc=00000001, cnonce=“0123456789abcdef”\r\n\r\n\r\n\r\n

  4. HTTP/1.1 200 OK の応答があることを確認
    それ以外のレスポンスコードが返る場合は、HTTP リクエストが正しいか確認ください。

  5. 続けて、音声通話が可能な場合はコンテンツタイプが返る
    Content-Type: audio/aac\r\n\r\n

    音声通話ができない場合はToo many usersが返る
    X-CurrentUsers: Too many users\r\n
    Content-Length: 0\r\n
    Content-Type: text/plain\r\n\r\n

    Too many users の応答があった場合、他のユーザによって既に双方向通話が利用されていることを示します。 直ちにSocketを切断し、他のユーザに通話を終了するよう依頼するか適当な時間の後に、再試行するなどしてください。

  6. Socket 通信は維持したまま、コンテンツタイプに合致する音声データを送る
    <132 bytes AAC Audio data>
    <132 bytes AAC Audio data>
    <132 bytes AAC Audio data>
    …
    ※短時間にあまり多くのデータを送ると音飛び等の原因になるため、33 バイトの倍数で区切ったものを数十ミリ秒間隔で送るなど、適宜調整してください。

  7. 通話を終了する場合は、Socket を切断します。

クラウド ¶

クラウド関連の設定を行います。本機能を以下ではクラウドモードと称します。

機能概要 ¶

検知の発生もしくは定期的な間隔で、指定のhttpサーバーへ以下の情報を送ることができる機能です。

・ライブ映像(リアルタイム映像)のアップロード(カメラからPUSH) *常時
・ライブ画像のアップロード(カメラからPOST)
・センサー情報のアップロード(カメラからPOST)
・イベント情報(検知種類,動画.画像)のアップロード(カメラからPOST)
・稼働状態の送信(カメラからPOST)
・設定の受信(カメラがGET)

  • 各種情報をサーバーに送信する前の一時領域としてSDカードを利用します。本機能のご利用にはSDカードを挿入してフォーマットを完了しておく必要があります。

  • 検知録画の保存先にNASを指定していた場合は、本機能を有効にした時点で保存先がSDカードに切り替わります。本機能が有効の間は保存先を変更することはできません。

使い方 ¶

  1. カメラにSDカードを挿入します。

  2. SDカードのフォーマットを完了します。

方法1: Web設定画面からSDカードのフォーマットを行います。
