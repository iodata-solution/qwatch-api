param.cgi ¶

POSTリクエストの場合、引数で与えられた値の反映結果をBodyにて返します。
無効な引数が指定された場合は、その引数については何も返しません。

リクエスト例 ¶

curl -vX POST --digest --user admin:1qaz http://192.168.1.8:80/camera-cgi/admin/param.cgi -d “action=update&Network_networkType=0&Network_IP=TEST&Network_netmask=255.255.255.0&Network_dummy=192.168.1.1”

Bodyの例 ¶

OK: Network_networkType=‘0’
ERROR: Network_IP=‘TEST’
OK: Network_netmask=‘255.255.255.0’

(Network_IPという引数に’TEST’という値は不正なのでERRORを返す。Network_dummyは無効な引数なので何も返さない)

その他 ¶

引数に問題なければ、実行結果をBodyにて返します。

リクエスト例 ¶

curl -vX GET --digest --user admin:1qaz http://192.168.1.8:80/camera-cgi/admin/reboot.cgi

Bodyの例 ¶

OK

audio/transmit.cgi  ¶

通話
GET/camera-cgi/audio/transmit.cgi

対象: TS-NS310W,TS-NS410W,TS-NA230WP
通話機能の音声を送信します。

Caution ¶

接続後、クライアントは応答のヘッダに含まれている文字列を確認してからBodyを送信してください。

<対応する音声形式>

Product AAC GSM
TS-NS310W Yes No
TS-NS410W Yes No
TS-NA230WP Yes No

<クライアントからの使用例>

  1. カメラの[HttpPort]にSocket接続

  2. Socket に次を送信

    GET /camera-cgi/audio/transmit.cgi HTTP/1.1\r\nConnection: Keep-Alive\r\nCache-Control: no-cache\r\n\r\n\r\n

  3. 401 Unauthorized の応答があった場合、Digest 認証情報を付与して再送

    GET /camera-cgi/audio/transmit.cgi HTTP/1.1\r\nConnection: Keep-Alive\r\nCache-Control: no-cache\r\nAuthorization: Digest username=“admin”, realm=“Network Camera”, nonce=3f16834beee735c795b53a4d25d5fa50, uri="/camera-cgi/audio/transmit.cgi", response=“d44876fe10cb98d9ee101ca527126412”, qop=“auth”, nc=00000001, cnonce=“0123456789abcdef”\r\n\r\n\r\n\r\n

    TS-NS310W Ver1.00.46またはTS-NS410W Ver1.02.18については、
    一度セッションを切断して再接続した後に再送する必要があります。

  4. HTTP/1.1 200 OK の応答があることを確認
    それ以外のレスポンスコードが返る場合は、HTTP リクエストが正しいか確認ください。

  5. 続けて、音声通話が可能な場合はコンテンツタイプが返る
    Content-Type: audio/aac\r\n\r\n

    音声通話ができない場合はToo many usersが返る
    X-CurrentUsers: Too many users\r\n
    Content-Length: 0\r\n
    Content-Type: text/plain\r\n\r\n

    Too many users の応答があった場合、他のユーザによって既に双方向通話が利用されていることを示します。 直ちにSocketを切断し、他のユーザに通話を終了するよう依頼するか適当な時間の後に、再試行するなどしてください。

  6. Socket 通信は維持したまま、コンテンツタイプに合致する音声データを送る
    <132 bytes AAC Audio data>
    <132 bytes AAC Audio data>
    <132 bytes AAC Audio data>
    …
    ※1度にあまり多くのデータを送ると音飛び等の原因になるため、33 バイトの倍数で区切ったものを数十ミリ秒間隔で送るなど、適宜調整してください。

  7. 通話を終了する場合は、Socket を切断します。

Example URI

GET http://192.168.1.2:80/camera-cgi/audio/transmit.cgi
Response  200
HideShow
Body
状態に応じて、いずれかの文字列を返します。
- `OK` - 音声データ受信の準備ができました。
- `Too many users` - 他のクライアントが通話中です。

状態が`OK`の場合、サポートする音声タイプをContent-Typeで返答します。
各Content-Typeの音声仕様は以下のとおり。
- `Content-Type: audio/aac-lc\r\n\r\n`
   - Container: Audio Data Transport Stream (ADTS)
   - Codec: AAC-LC
   - Channel: 1ch
   - Bitrate: Max.64kbps
   - Sampling rate: 8kHz (TS-NS410W Ver1.02.18以降,TS-NA230WPは16kHz)<BR>
- `Content-Type: audio/aac\r\n\r\n`
   - Container: Audio Data Transport Stream (ADTS)
   - Codec: AAC-LC
   - Channel: 1ch
   - Bitrate: Max.64kbps
   - Sampling rate: 16kHz
- `Content-Type: audio/gsm\r\n\r\n`
   - Codec: GSM 06.10
   - Channel: 1ch
   - Bitrate: 13.2kbps
   - Sampling rate: 16kHz

クラウド ¶

対象: TS-NS410W,TS-NA230WP

Caution ¶

各種情報をサーバーに送信する前の一時領域としてSDカードを利用します。クラウドモードのご利用にはSDカードを挿入してフォーマットを完了しておく必要があります。

機能概要 ¶

検知の発生もしくは定期的な間隔で、指定のhttpサーバーへ以下の情報を送ることができる機能です。

・[ライブ映像のアップロード](カメラからRTMP PUSH) *常時
・[ライブ画像のアップロード](カメラからPOST)

・[イベント情報(検知種類,動画.画像)のアップロード](カメラからPOST)
・[稼働状態の送信](カメラからPOST)
・[設定の受信](カメラがGET)

対応バージョン ¶

Product ファームウェア
TS-NS410W Ver1.05以降
TS-NA230WP Ver1.03以降

※出荷時点でクラウドモードに対応していない古いファームウェアのものがあります。最新のファームウェアにアップデートのうえご利用ください。

使い方 ¶

  1. カメラにSDカードを挿入します。

  2. SDカードのフォーマットを完了します。

方法1: Web設定画面からSDカードのフォーマットを行います。
方法2: 本仕様書記載の[SDカードのフォーマット]APIを実行します。
方法3: 同一機種の他のカメラ個体にて事前にフォーマットを済ませたSDカードが利用可能です。

  1. 利用する各機能のAPI[ライブ映像のアップロード][ライブ画像のアップロード][イベント情報(検知種類,動画.画像)のアップロード][稼働状態の送信][設定の受信]にて設定を行います。

  2. [クラウドサーバーURL]を設定します。クラウドモードが開始します。

3,4は各機能ごとに個別にリクエストするほか、まとめて同時にひとつのリクエストで行っても構いません。

  1. クラウドモードを解除する場合は、[クラウドサーバーURL]のCloud_cloudserver_urlをnullに設定します。

httpアップロード機能 ¶

– 当社確認環境
次の環境で動作の確認を行いました。

nginx + nginx-rtmp-module + Puma + Rails

上記の環境は一例です。
特に指定はありません。ご都合のよいWebサーバーをご利用いただけます。

– 認証
サーバーへの接続に認証を設けることができます。

  • BASIC認証

  • ユーザ名: admin + MACアドレスの下6桁(英数字大文字)
    (例: MACアドレスが34:76:c5:ab:cd:efの場合、ユーザ名はadminABCDEFになります)

  • パスワード: カメラに設定したadminパスワード

– ディレクトリ構造
サーバー側は各機能によるアップロード先ディレクトリ([/Picture][/sensor][/event][/status]にてPOSTを受付けできる必要があります。

カメラからhttpサーバーに送信(POST)される情報とパス:

  • ライブ画像のアップロード
    http://[Cloud_cloudserver_url]/Picture/YYYY-MM-DD-hh:mm:ss-Image.jpg

  • イベント情報(検知種類,動画.画像)のアップロード
    http://[Cloud_cloudserver_url]/event/YYYY-MM-DD-hh:mm:ss.[jpg|ts|m3u8|json]
    動作検知などで撮影された静止画、動画(HLS形式)、検知の種類(json形式)がアップロードされます。

  • 稼働状態の送信
    http://[Cloud_cloudserver_url]/status/ipcamStatus.xml カメラの設定値などがxml形式でアップロードされます。

カメラがhttpサーバーから受信(GET)する情報とパス:

  • 設定の受信(カメラがGET)
    http://[Cloud_cloudserver_url]/serverConfig.xml

– その他条件

  • サーバーからのHTTPステータスコード 300番台には非対応です。

  • 定期処理系の機能[ライブ画像のアップロード][稼働状態の送信][設定の受信]は、サーバーへのHTTP GET/POSTに失敗してもリトライは行いません。設定した送信/受信間隔の次のタイミングをお待ちください。

詳細は各機能の項目を参照ください。

稼働状態の送信
POST/camera-cgi/admin/param.cgi{?action,Cloud_StatusReport_enable,Could_StatusReport_interval}

対象: TS-NS410W,TS-NA230WP
ipcamStatus.xmlを指定された時間間隔でアップロードするための設定を行います。

設定情報は以下のアドレスにPOSTされます。
http://[Cloud_cloudserver_url]/status/ipcamStatus.xml

※取得できるxmlの各要素の詳細はそれぞれ対応するAPIのParameterにてご確認ください(API仕様書に記載の無い値はサポート対象外です)。

Example URI

POST http://192.168.1.2:80/camera-cgi/admin/param.cgi?action=update&Cloud_StatusReport_enable=1&Could_StatusReport_interval=90

対象: TS-NS410W,TS-NA230WP

http://[Cloud_cloudserver_url]/serverConfig.xml

※上記URLに配置するserverConfig.xmlの例は下記のXML Sampleを参照ください。
